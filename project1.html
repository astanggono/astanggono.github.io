Song Predictor Rating

The R script is an integral part of a project aimed at predicting the popularity ratings of approximately 20,000 popular songs based on their auditory features. The core objective of this project is to develop a robust predictive model that can accurately forecast the popularity ratings of songs using a comprehensive set of song-specific characteristics.

The R script leverages several libraries, including tidyr, mice, and dplyr, to facilitate the data analysis and modeling process. Data cleaning and preprocessing are vital steps in preparing the datasets for modeling. Both the analysis and scoring datasets undergo similar data cleaning steps.

Firstly, the data is read and loaded into the script. Then, data cleaning commences. Missing values in the datasets are imputed using the mice package with the "cart" method. This helps ensure that the datasets are complete and suitable for analysis.

To handle the 'genre' column, which contains categorical data, a series of data transformations are applied. These transformations involve removing square brackets and single quotation marks, splitting the column into separate rows based on commas, and removing special characters and excess white spaces. Additionally, a new variable, 'track_explicit,' is created as a binary indicator based on the 'track_explicit' column.

For modeling, a random forest regression model (forest_ranger) is employed. This model is chosen due to its capability to handle complex relationships between predictors and the target variable, song ratings. It includes a wide range of predictors, such as 'pop,' 'acousticness,' 'rock,' and various other features that can influence song ratings.

To evaluate the model's performance, Root Mean Squared Error (RMSE) is calculated for both the training and testing datasets. The RMSE serves as a measure of how well the model predicts song ratings.
