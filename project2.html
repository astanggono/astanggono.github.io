Competition Overview (Group Project)

In this group competition, the objective is to collaboratively build machine learning models for image recognition using the MNIST Fashion database, which contains images of different types of apparel. The goal is to predict the category label of these images based on their pixel brightness values. The images are represented as grids of pixels, and the data has been condensed to 49 pixels (7x7) per image for computational tractability. The challenge involves finding the best machine learning models, determining the sample size required for accurate predictions, and optimizing the computational time.

The Data

  • The dataset includes grayscale images of apparel, with pixel values ranging from 0 (white) to 255 (black).
  • The data have been condensed from images of 784 (28x28) pixels into 49 pixels (7x7).
  • Two data files are provided: training data and testing data.
  • Variables include the label (category) and pixel_1 to pixel_49, representing grayscale measurements for each pixel.

The Challenge

Participants, working in groups, must build and evaluate different machine learning models on various sample sizes. The quality of models, their running times, and sample sizes are evaluated using a scoring function: Points = 0.15 * A + 0.1 * B + 0.75 * C. A represents the proportion of training data used, B is a penalty for long-running algorithms, and C is the proportion of incorrectly classified predictions.

Rules

  • Participant groups must build 10 different classification models using at least 5 specified techniques, including logistic regression, K-Nearest Neighbors, decision trees, random forests, ridge/lasso/elastic net regression, support vector machines, gradient boosted regression models, and neural networks.
  • Multiple variants of the same algorithm with different parameter settings count as separate models.
  • Participant groups must select three different sample sizes and create three model development sets for each size.
  • Each model is applied to all model development sets, resulting in a total of 90 models.

Evaluation

Participant groups compute A, B, C, and Points for each model and sample size. Averages are calculated across the three model development sets. The final scoreboard displays average results for 30 combinations of Model and Sample Size, sorted by increasing Points.

The group competition challenges participants to work collaboratively to find the best models, optimize sample sizes, and minimize computational time while achieving accurate predictions on the testing data. The Points metric reflects the overall quality of the models developed by the participant groups.
